¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es una red que tiene muchas computadoras conectadas entre sí. Mientras que el World Wide Web es un sistema de información que almacena páginas dentro de esta red.

¿Cuáles son las partes de un URL?

Una url cuenta con:
  • Protocolo
  • Subdominio
  • Dominio

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET - Recupera un archivo de algún servidor web para leer.
  • HEAD - Igual al GET pero este solicita los metadatos de un archivo, en cambio de todo el elemento.
  • POST - Envía datos e información a un recurso web del servidor.
  • PUT - Crea o carga un nuevo recurso al servidor. Si ya existe, reemplaza el que existe con el recurso que se carga.
  • PATCH - Aplica modificaciones parciales a un recurso específico.
  • DELETE - Solicita al servidor web que se borre algún recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST, porque es el que se encarga de enviar datos a un recurso web.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, porque este error sucede cuando se accede a algo prohibido, hay un enlace roto, defectuoso o que ya no existe y que, por lo tanto, ya no se puede navegar en él.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, esto sucede cuando algo sale mal en el servidor del sitio web. Por lo que el desarrollador tendrá que arreglar el servidor del sitio web.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Algunas diferencias entre HTML 4 y HTML 5 son:
  • Sintaxis simplificada en HTML5
  • Elementos como header y footer

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de un sistema de información es
  1. Nacimiento
  2. Desarrollo
  3. Operación
  4. Mantenimiento
  5. Muerte

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de un sistema de información es
  1. Planificación
  2. Análisis de requerimientos
  3. Diseño
  4. Desarrollo
  5. Integración y pruebas
  6. Implementación
  7. Mantenimiento

CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Que se use únicamente cuando se quiera aplicar una regla sobre las demás. Es decir, el uso de !important lo que hará es que ignorará las reglas subsecuentes y solo aplicará la regla en la que se declaró

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Porque ésta puede interferir con el texto y los elementos que tenga la página. Puede ser que esto le afecte la vista o sea difícil de comprender para el usuario.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Para elegir las unidades de una propiedad es importante tener en mente si queremos a un elemento en un específico lugar sin importar el tamaño de la ventana, o si el tamaño si importará para definirlo.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Porque minimiza el tiempo en el que procesa el código y el ancho de banda. Esto hace que mejore inmensamente la rapidez y accesibilidad para que el usuario tenga una mejor experiencia.
